# Yahoo Finance 爬虫优化方案

## 当前状态
- yfinance 1.2.0 + curl_cffi 0.13.0 ✅ 已恢复
- 40只股票批量下载约20秒，每30秒一次

## 优化清单

### 1. 缓存分层（减少请求频率）
- 盘前/盘后：每5分钟抓取一次（波动小）
- 正常盘：每30秒（当前策略）
- 周末/假日：完全跳过

### 2. 分批请求（降低单次延迟）
- 将40只股票分2批，每批20只
- 两批间隔5秒，避免连续请求触发限流
- 预估总延迟从20秒降至12秒（并行拉取）

### 3. 增量更新（仅抓取变化的）
- 维护价格缓存，仅对价格变动>0.1%的股票触发重抓
- 减少80%的无效请求

### 4. 备用数据源（故障切换）
- Alpha Vantage（免费500次/天）
- Polygon.io（付费，低延迟）
- 已在config/settings.py预留配置

### 5. 请求头伪装池
- 轮换User-Agent列表
- 随机化请求间隔（30s ± 5s抖动）
