## 发现的问题

### 🔴 致命：基准曲线始终返回0数据
- **根因**：`BenchmarkTracker.initialize_from_history()` / `fetch_nasdaq_history()` 从未在 `web_server.py` 中调用
- **影响**：`/api/benchmark_curve` 端点永远返回0个数据点，前端"策略 vs 纳斯达克"图表空白
- **修复**：在服务启动（状态恢复后）自动调用 `_benchmark.fetch_nasdaq_history('6mo')`
- **验证**：重启后返回124个历史数据点，纳指收益率13.77%

### 🔴 致命：反序列化后 `_nasdaq_equity_dict` 丢失
- **根因**：`deserialize_benchmark()` 仅恢复 `nasdaq_equity_curve` Series，未恢复 `_nasdaq_equity_dict` 字典
- **影响**：重启后100个tick内，新数据会覆盖所有历史纳指曲线数据
- **修复**：`deserialize_benchmark()` 同步恢复 `_nasdaq_equity_dict` 和 `_strategy_equity_dict`

### 🟡 严重：`_strategy_equity_dict` 未初始化
- **根因**：`BenchmarkTracker.__init__()` 未声明 `self._strategy_equity_dict`
- **影响**：如调用 `append_strategy_snapshot()` 或 `_ensure_curves_synced()` 会抛出 AttributeError
- **修复**：在 `__init__` 添加 `self._strategy_equity_dict: dict = {}`

### 🟡 严重：`update()` 中策略曲线和纳指曲线更新方式不一致
- **根因**：纳指用 dict 存储后每100次同步到 Series；策略直接 `pd.concat` 追加
- **影响**：策略曲线实时更新但纳指曲线滞后100个tick，`_ensure_curves_synced()` 形同虚设
- **修复**：统一使用 dict 存储，`update()` 中调用 `_ensure_curves_synced()` 同步
- **附加**：`api_benchmark_curve` 和 `build_status_data` 读取前调用 `_ensure_curves_synced()`

### 🟡 严重：Yahoo时区bug导致 `initialize_from_history` 失败
- **根因**：Yahoo返回 tz-aware 时间戳，与 `pd.Timestamp(start_date)` (tz-naive) 比较报错
- **修复**：在 `initialize_from_history` 中添加 `df.index.tz_localize(None)` 去除时区

### 🟡 严重：K线加载失败无用户提示
- **根因**：`loadKlineData()` 异常时只 `console.error`，用户看不到任何反馈
- **修复**：添加 `klineStatus` 元素，显示"⏳ 加载中..." / "✅ N 根K线" / "⚠️ 无K线数据" / "❌ 加载失败"

### 🟢 轻微：基准曲线空数据无前端提示
- **修复**：添加 `benchStatus` 元素，显示"⏳ 等待交易数据..."

## 补充修复 (13:50) — 图表切换后不显示

### 🟡 严重：切换面板后K线图和基准曲线消失
- **根因**：面板 `.panel{display:none}` 切换到 `.panel.active{display:block}` 时，CSS 有 0.25s fadeIn 动画
  `requestAnimationFrame` 在容器完成布局前就触发了 `chart.resize()`，此时容器 offsetWidth=0
- **影响**：切换到"总览"再切回"K线图"或"分析"，图表不复显示
- **修复**：新增 `safeResize(chart, containerId, maxRetries, delay)` 函数
  - 默认 150ms 延迟后开始尝试
  - 检查容器 `offsetWidth > 0 && offsetHeight > 0`
  - 不满足则最多重试 10 次，每 150ms 一次
- **验证**：`curl localhost:8080` 确认 HTML 包含 `safeResize` 函数（3处引用）
