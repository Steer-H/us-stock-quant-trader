# 交易指令格式規範

> 系統: 美股量化交易系統  
> API: `GET /api/signals`  
> 更新: 2026-06-13

---

## API 響應結構

```json
{
  "signals_md": "Markdown格式文本，每行一條指令",
  "signals": [
    {
      "time": "09:30:01",
      "action": "BUY",
      "ticker": "AAPL",
      "qty": 50,
      "price": 175.00,
      "reason": "開盤建倉"
    }
  ]
}
```

---

## 單條指令格式

```
**{emoji} {action}** {ticker} {qty}股 @ ${price} | {time} | {reason}
```

### 欄位說明

| 欄位 | 類型 | 說明 | 示例 |
|------|------|------|------|
| `emoji` | str | 🟢買入 / 🔴賣出 | `🟢` |
| `action` | str | BUY 或 SELL | `BUY` |
| `ticker` | str | 美股代碼 | `AAPL` |
| `qty` | int | 股數 | `50` |
| `price` | float | 成交單價(USD, 2位小數) | `175.00` |
| `time` | str | 執行時間 HH:MM:SS | `09:30:01` |
| `reason` | str | 觸發條件 | `開盤建倉` |

---

## 六種交易原因

| 原因 | 觸發條件 | 概率 |
|------|----------|------|
| `開盤建倉` | 美東 09:30 首次持倉配置 | 100% |
| `短線止盈({x}%)` | 浮動盈虧 ≥ +3% | 80% |
| `短線止損({x}%)` | 浮動盈虧 ≤ -4% | 90% |
| `ML預測賣出` | 盈虧 < 1% 且模型信號偏空 | 17.5% |
| `超時平倉({x}分鐘)` | 單筆持倉 > 30分鐘 | 100% |
| `ML短線買入` | 動量信號 + 現金 > $5000 | 按信號 |

## 儀錶盤渲染

指令卡片樣式：
- 字體: 等寬 (`SF Mono / Monaco / Menlo`)
- 字號: 12px
- 🟢 BUY 行: 綠色加粗
- 🔴 SELL 行: 紅色加粗
- 股票代碼: 白色加粗
- 價格/時間/原因: 灰色輔助文字
- 滾動: 最大高度 300px，超出可滾動
- 刷新: 每 3 秒自動拉取
